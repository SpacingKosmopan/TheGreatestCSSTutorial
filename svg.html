SVG (Scalable Vector Graphics)

is an XML-based vector image format used directly inside HTML. Unlike regular images (JPG, PNG), SVG never loses quality when zoomed in and maintains an incredibly small file size.

The Main Container

Every SVG graphic must be wrapped inside the root < svg > tag. Its key attributes control how the image scales:
width / height: Sets the physical display size on the web page.

Basic Shape Tags and Applications

These structural elements build your visual components:

  • <rect> (Rectangle): Draws squares and rectangles. Can have rounded corners.
    Syntax: <rect x="X_COORD" y="Y_COORD" width="WIDTH" height="HEIGHT" rx="RADIUS_X" ry="RADIUS_Y" />
  • <circle> (Circle): Renders perfect circles using a center point and a radius.
    Syntax: <circle cx="CENTER_X" cy="CENTER_Y" r="RADIUS" />
  • <ellipse> (Ellipse): Renders oval shapes using distinct horizontal and vertical radii.
    Syntax: <ellipse cx="CENTER_X" cy="CENTER_Y" rx="RADIUS_X" ry="RADIUS_Y" />
  • <line> (Line): Connects two distinct coordinate points with a straight line.
    Syntax: <line x1="START_X" y1="START_Y" x2="END_X" y2="END_Y" stroke="COLOR" />
  • <polyline> (Polyline): Connects a series of straight lines together (ideal for open shapes like charts).
    Syntax: <polyline points="X1,Y1 X2,Y2 X3,Y3" fill="none" stroke="COLOR" />
  • <polygon> (Polygon): Connects a series of points and automatically closes the shape (ideal for triangles and stars).
    Syntax: <polygon points="X1,Y1 X2,Y2 X3,Y3" />
  • <path> (Path): The ultimate freestyle drawing tool. Uses specialized text commands to draw complex, organic curves and custom shapes.
    Syntax: <path d="M X Y L X Y C X1 Y1, X2 Y2, X Y Z" />
  • <text> (Text): Renders searchable, selectable, and SEO-friendly typography directly inside the graphic.
    Syntax: <text x="START_X" y="BASELINE_Y">Your Text Here</text>

Drawing commands

M / m - Move to (Lift and Place the Pen)

This command lifts the pen off the digital paper and moves it to a new starting point. It does not draw anything. Every path must start with an M command.
Syntax: M x y (Absolute) or m dx dy (Relative)

L / l - Line to (Draw a Straight Line)

This command draws a straight line from the pen's current position to the new target position.
Syntax: L x y (Absolute) or l dx dy (Relative)

Z / z - Close Path (Return Home)

This command snaps a straight line from your current position back to the very first point of the current sub-path (where you last used the M command).
Syntax: Z or z (Case does not matter; it takes no coordinate numbers)
Example: M 10 80 L 50 20 L 90 80 Z
What it does: It draws a line from (10,80) to (50,20), then down to (90,80). Finally, Z automatically draws a line from (90,80) back to the start at (10,80), perfectly closing a triangle.

C / c - Cubic Bézier Curve (Smooth Curves)

This is the most powerful command. It draws a curved line using two external control points that pull the line like virtual magnets, creating organic shapes.
Syntax: C x1 y1, x2 y2, x y

  • x1 y1: Control point 1 (controls the curve's direction leaving the start point)
  • x2 y2: Control point 2 (controls the curve's direction entering the end point)
  • x y: The final destination where the line actually ends

viewbox

Bez viewBox (<svg width="100" height="100">): Przeglądarka tworzy sztywne okno o wymiarach 100x100 pikseli. Jeśli Twój kształt ma współrzędną X=150, zostanie on bezwzględnie ucięty, bo wystaje poza te 100 pikseli.

Z viewBox (<svg width="100" height="100" viewBox="0 0 200 200">): Mówisz przeglądarce: "Mój rysunek ma wewnętrzny rozmiar 200x200, ale ściśnij go tak, aby zmieścił się w fizycznym oknie 100x100". Kształt o współrzędnej X=150 będzie idealnie widoczny (zostanie przeskalowany do 75 pikseli).

Styling

Fill

/* Kontrola wnętrza kształtów */
selector {
  fill: #3498db;            /* Kolor wypełnienia (nazwa, HEX, RGB) */
  fill: url(#id-gradientu); /* Wypełnienie gradientem lub wzorem z <defs> */
  fill: none;               /* Brak wypełnienia (przezroczyste wnętrze) */
  fill-opacity: 0.5;        /* Przezroczystość wypełnienia (od 0.0 do 1.0) */
  fill-rule: nonzero;       /* Reguła wypełniania: domyślna dla skomplikowanych ścieżek */
  fill-rule: evenodd;       /* Alternatywna reguła wypełniania */
}

Stroke (line and border)

/* Kontrola konturów i linii */
selector {
  stroke: #e74c3c;          /* Kolor linii (nazwa, HEX, RGB, gradient) */
  stroke: none;             /* Brak obramowania */
  stroke-width: 3px;        /* Grubość linii (w pikselach lub bez jednostek) */
  stroke-opacity: 0.8;      /* Przezroczystość linii (od 0.0 do 1.0) */
  
  /* Przerywane linie i animacje */
  stroke-dasharray: 5, 5;   /* Wzór linii przerywanej: [długość kreski, długość przerwy] */
  stroke-dasharray: 10, 5, 2, 5; /* Złożony wzór kreskowania */
  stroke-dashoffset: 45;    /* Przesunięcie wzoru kreski (kluczowe do animacji rysowania) */
  
  /* Wykończenie krawędzi */
  stroke-linecap: butt;     /* Płaskie, ucięte końce linii (domyślne) */
  stroke-linecap: round;    /* Zaokrąglone końce linii */
  stroke-linecap: square;   /* Kwadratowe końce linii (wydłużone o grubość konturu) */
  
  /* Łączenie linii w rogach */
  stroke-linejoin: miter;   /* Ostre, spiczaste narożniki (domyślne) */
  stroke-linejoin: round;   /* Zaokrąglone narożniki */
  stroke-linejoin: bevel;   /* Ścięte narożniki */
  stroke-miterlimit: 4;     /* Limit ostrości narożnika miter zanim przejdzie w bevel */
}

Text

/* Formatowanie napisów w tagach <text> i <tspan> */
selector {
  font-family: 'Arial', sans-serif;
  font-size: 16px;
  font-weight: bold;
  letter-spacing: 1px;
  word-spacing: 2px;
  text-decoration: underline;
  
  /* Pozycjonowanie i wyrównanie tekstu (specyficzne dla SVG) */
  text-anchor: start;       /* Wyrównanie do lewej względem punktu X */
  text-anchor: middle;      /* Wyśrodkowanie tekstu względem punktu X */
  text-anchor: end;         /* Wyrównanie do prawej względem punktu X */
  
  dominant-baseline: auto;            /* Domyślna linia bazowa */
  dominant-baseline: central;         /* Wyrównanie pionowe do środka geometrycznego */
  dominant-baseline: mathematical;    /* Wyrównanie do osi matematycznej */
  
  writing-mode: horizontal-tb; /* Tekst poziomy */
  writing-mode: vertical-rl;   /* Tekst pionowy od prawej do lewej */
}

Visibility, effects

/* Kontrola renderowania, przezroczystości i filtrów */
selector {
  opacity: 0.7;             /* Przezroczystość całego elementu (wypełnienie + linia) */
  visibility: hidden;       /* Ukrycie elementu (ale zachowuje miejsce w strukturze) */
  display: none;            /* Całkowite usunięcie elementu z widoku */
  
  filter: url(#id-filtra);  /* Nałożenie filtra SVG (np. rozmycia, cienia z sekcji <defs>) */
  
  /* Zarządzanie renderowaniem (optymalizacja jakości vs wydajność) */
  shape-rendering: auto;               /* Domyślny balans przeglądarki */
  shape-rendering: optimizeSpeed;      /* Wyłącza antyaliasing (ostre krawędzie, szybkie) */
  shape-rendering: crispEdges;         /* Wymusza ostre krawędzie bez rozmyć (dobre dla linii) */
  shape-rendering: geometricPrecision; /* Maksymalna jakość wygładzania krawędzi */
  
  /* Zarządzanie renderowaniem tekstu */
  text-rendering: optimizeLegibility;  /* Priorytet dla czytelności tekstu */
  
  /* Kolorowanie obiektów wektorowych na podstawie koloru tekstu */
  color: #ff0000;
  fill: currentColor;       /* Pobierze kolor z właściwości 'color' (przydatne przy ikonach) */
}

Pointer events

/* Kontrola reagowania na kliknięcia i najechania myszką */
selector {
  pointer-events: none;         /* Element ignoruje myszkę (kliknięcia przelatują "na wylot") */
  pointer-events: auto;         /* Domyślne zachowanie */
  pointer-events: fill;         /* Reaguje tylko na kliknięcie wewnątrz kształtu */
  pointer-events: stroke;       /* Reaguje tylko na kliknięcie dokładnie w linię */
  pointer-events: visiblePainted;/* Reaguje na wszystko, co jest widoczne i pokolorowane */
}
